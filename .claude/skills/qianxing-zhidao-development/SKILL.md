## 二、赛题对齐原则

必须严格围绕赛题三「围绕山地旅游游客兴趣建模与个性化导览」，体现四个核心方向：

1. **游客兴趣建模与个性化导览** — 识别兴趣、人群、体力偏好、节奏，生成个性化推荐
2. **山地动态路径规划与实时重规划** — 体力消耗、景点热度、天气、拥堵模拟 + 替代方案
3. **游客异常行为识别与风险预警** — 偏离路线、长时间停留、体力下降、高风险区域
4. **游客求助与安全辅助** — 附近服务点、医疗点、救援点、紧急联系、安全提醒

当前阶段用规则算法和模拟数据实现，后续接入真实能力。

## 三、评审导向原则

后续开发必须服务于四个评审要点：

1. **创新性** — 体现 AI、画像、推荐、重规划、安全预警，不做普通旅游网站
2. **技术难度** — 清晰算法逻辑、数据结构、风险评估规则、评分机制
3. **完成度** — 小程序能走通完整流程（首页→兴趣→画像→推荐→详情→伴游→安全→行程）
4. **应用价值** — 贴近贵州山地旅游真实痛点（路线规划难、体力分配难、安全风险高、民族文化讲解不足）

## AI 能力模拟规范

当前阶段所有AI能力通过规则引擎模拟：

1. **画像生成：** 根据用户选择的兴趣标签，用加权规则生成游客画像
2. **路线推荐：** 基于画像与路线的标签匹配度进行打分排序（标签60分+体力15+人群15+天数10）
3. **AI伴游问答：** 从知识库中匹配最相关的问答对，加随机延迟模拟思考
4. **安全守护：** 规则风险评分（体力预警+偏离检测+停留超时+天气提醒）
5. **文案风格：** AI生成文案应模仿"AIGC风格"，正式、有条理、有数据感

## UI 设计铁律（Web + 小程序通用，强制执行）

### 动画：强制使用 GSAP

- Web 端：`import gsap from 'gsap'`（已安装），每个页面必须有入场动画
- 小程序端：`wx.createAnimation` 实现等效效果
- 卡片列表：stagger 依次出现
- 页面切换：过渡动画
- 禁止 CSS `@keyframes` 作为主要动画手段

### 资源：图片和视频优先

- **优先级：真实图片/视频 > SVG 图标 > 纯文字**
- **禁止 Emoji 图标**（🧭🏞️🤖 等不允许出现在正式页面上）
- 功能图标使用 Element Plus Icons（Web）或 SVG（小程序）
- 景点展示必须使用真实照片
- 图片缺失时：CSS 渐变占位 + 立即向用户索要

### 色彩

- 主色：`#1f8f5f`（翠绿 · 贵州山水）
- 辅色：`#2f6bff`（科技蓝 · AI智能）
- 强调：`#e67e22`（暖橙 · 苗族元素）
- 背景：`#f4f8f5`（浅绿灰）
- 不引入第四种主题色

### 美观度标准

- 大留白 + 清晰信息层级，字重对比明显（标题 700+ / 正文 400）
- 移动端按钮高度 ≥ 44px（Apple HIG）
- 图片必须高清（Web ≥ 1920px，小程序 ≥ 750px）
- 毛玻璃卡片、渐变光影、圆角 12-30px

```
1. 输出开发计划

   - 为什么这样划分
   - 本次目标（一句话）
   - 涉及文件（新建/修改列表）
   - 是否涉及用户可见文案
   - 是否需要清理 Demo/测试/比赛类词汇
   - 是否贴合赛题三的四个核心方向
   - 未来后端接口预留
   - 验证方式

2. 等待确认
   - 用户回复"确认"或"继续"后进入第3步

3. 编写代码
   - 先创建类型定义（如需要）
   - 再创建数据（如需要）
   - 再创建组件/页面
   - 最后配置路由

4. 验证
   - 运行项目确认不报错
   - 确认页面可正常访问
   - 确认与已有页面无冲突
   - 确认用户端无违禁词汇
```

## 禁止事项

1. 不要在用户端出现 Demo、测试、比赛、开发中、占位等词
2. 不要做普通旅游攻略 — 必须体现 AI 伴游 + 安全守护
3. 不要只做路线推荐 — 必须逐步加入安全守护能力
4. 不要删除现有 Web 成果
5. 不要破坏小程序主流程
6. 不要引入复杂依赖（除非经过确认）
7. 不要在小程序启动页一次性加载大量数据
8. 不要把 Web 写法带入小程序
9. 不要为了炫技牺牲可运行性 — 每次修改后项目必须仍能正常运行
10. 不要偏离贵州山地旅游场景
11. 禁止一次性大规模修改 — 每次只完成一个小步骤
12. 禁止跳过类型定义 — 所有数据必须先有TypeScript类型
13. 未经确认不写代码 — 先输出计划，等待确认后再实施
14. 统计展示只用纯组件 — 禁止 ECharts
15. 不使用 Emoji 代替图标或景点图
