# 跨项目对话记忆技能

## 触发方式

- 用户输入 `/conversation-memory` 或 `/对话记忆` 或 `/memory`
- 用户说"记住这次对话"、"记录一下"、"保存对话"等
- 在每次有意义对话结束时主动记录（完成实质性开发、修bug、做决策后）

## 技能职责

这个技能让你可以在**任意文件夹/项目**中：
1. **记录**：保存每次对话的关键信息（做了什么、改了哪些文件、为什么这样决策）
2. **检索**：下次用户提到某次对话时快速定位
3. **清理**：对话超过 50 次后自动删除最旧的记录

---

## 一、记录对话

### 什么时候记录

在以下情况，**主动**询问用户是否需要记录：
- 完成了一个功能开发或 bug 修复
- 做出了重要的架构/设计决策
- 用户说"今天就到这里"或类似结束语
- 跨越了多个 Phase 的大批量修改完成

### 记录的存储位置

```
C:\Users\27327\.reasonix\conversation-memory\
  INDEX.md              ← 总索引，每行一条记录
  conversations\        ← 每条对话一个 .md 文件
    2026-05-15-001.md
    2026-05-15-002.md
    ...
```

### 记录的文件命名

`YYYY-MM-DD-NNN.md`，其中 NNN 是当天序号（001, 002, ...）。

检查当天已有记录数量后自增。

### 记录文件格式

每个对话文件使用以下 Markdown 格式：

```markdown
---
date: YYYY-MM-DD
project: qianxing-zhidao
tags: [前端, Vue3, 功能开发]
---

## 做了什么

（2-5 句话概括本次对话完成的主要工作）

## 修改的文件

- `path/to/file1.vue` — 简述改了什么
- `path/to/file2.ts` — 简述改了什么

## 关键决策

- 决策 1 及原因
- 决策 2 及原因

## 待续事项

- 未完成的 TODO（如果有）
- 下次可以继续的方向
```

### 更新 INDEX.md

每次记录后，在 INDEX.md **最前面**插入一行：

```markdown
- [YYYY-MM-DD NNN](conversations/YYYY-MM-DD-NNN.md) — 简短描述（< 80 字符）
```

---

## 二、检索对话

1. 先读 `INDEX.md`，根据关键词匹配
2. 再读对应的 `conversations/YYYY-MM-DD-NNN.md` 获取详情
3. 如果 INDEX.md 匹配不到，用 Grep 搜索 conversations 目录

---

## 三、清理旧记录

当 INDEX.md 中有效记录超过 **50 条**时：
1. 按日期从旧到新排序
2. 删除最旧的 N - 50 条记录对应的 `.md` 文件
3. 从 INDEX.md 中移除对应行
4. 保留最新的 50 条

标注了 `pinned: true` 的记录跳过不删除。

---

## 四、注意事项

- **简洁优先**：每个对话文件不超过 50 行，只记关键信息
- **不记代码细节**：代码在 git 里，不在记忆里。记忆记的是 WHY 和 WHAT，不是 HOW
- **tag 要具体**：如 `[Vue3, 小程序, bug修复, 配置]`，方便检索
